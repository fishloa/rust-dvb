# Annex C — MPEG-2 CRC-32

**Spec:** ETSI EN 300 468 v1.19.1 Annex C (informative) + ISO/IEC 13818-1 Annex B
**Implementation:** `crates/dvb_si/src/crc.rs`

## Purpose

All SI and PSI sections carry a 32-bit CRC at the end of the section. The CRC covers all
bytes from the start of the section (including `table_id`) through the byte immediately
preceding `CRC_32`. Receivers validate the CRC before processing a section; invalid sections
are silently discarded.

## Polynomial

Generator polynomial: `G(x) = x^32 + x^26 + x^23 + x^22 + x^16 + x^12 + x^11 + x^10 + x^8 + x^7 + x^5 + x^4 + x^2 + x + 1`

Hex representation of the remainder: `0x04C11DB7`

The CRC is initialized to `0xFFFFFFFF`. The final CRC value (after processing all bytes) should
be `0x00000000` when the `CRC_32` field is included in the input data (self-checking form).

## Algorithm

```rust
pub const CRC32_TABLE: [u32; 256] = /* generated by const fn build_table() */;

pub fn crc32(data: &[u8]) -> u32 {
    let mut crc: u32 = 0xFFFF_FFFF;
    for &byte in data {
        let idx = ((crc >> 24) ^ (byte as u32)) & 0xFF;
        crc = (crc << 8) ^ CRC32_TABLE[idx as usize];
    }
    crc
}

pub fn verify(section_bytes: &[u8]) -> bool {
    // section_bytes includes CRC_32 field at the end
    crc32(section_bytes) == 0
}
```

## Encoding note

The `CRC_32` field is transmitted `rpchof` (remainder polynomial coefficients, highest order first),
meaning the MSB of the 32-bit CRC is transmitted first.

## Cross-references

- Used by: every table that has `CRC_32` in its syntax (NIT, BAT, SDT, EIT, TDT, TOT, RST, DIT, SIT, PAT, PMT, CAT, SAT)
- ST (Stuffing Table) does NOT carry CRC_32
- Reference: ISO/IEC 13818-1 Annex B

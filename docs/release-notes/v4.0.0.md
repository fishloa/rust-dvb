# 4.0.0 — section vs. complete table

A deliberate `dvb-si` API break that finally separates **one wire section** from
a **complete logical table**, and adds the layer that assembles the latter from
the former.

## Why

3.x stopped at single-section decode. Its parsers were *named* for tables
(`Nit`, `Sdt`, `AnyTable`) but each returned exactly one PSI/SI section. A
consumer that did `pop_section() → Nit::parse() → dedup by version_number` kept
only the **first** section of a multi-section table and silently dropped the
rest — every section of a sub-table shares one `version_number`. On Astra
19.2°E this lost ~half the transponders of a multi-section NIT_actual and, via
NIT_other, entire other networks; BAT bouquets showed one service instead of
their full membership (#32). Every consumer of NIT/BAT/SDT/EIT needs the *whole*
table, never a fragment — so 4.0 centralizes the assembly.

## What changed

**Names now tell the truth.** Single-section parsers gain a `Section` suffix
(`NitSection`, `SdtSection`, `PatSection`, …), the dynamic dispatcher is
`AnyTableSection`, `SectionEvent::table()` is `table_section()`, and `TableId`
variants are CamelCase (`TableId::Pat`, `TableId::MpeFec`). There are no
compatibility aliases — update imports and matches directly.

**New `collect` module — the section → complete-table rung.**

- `SectionSetCollector` — feed section bytes; get `Some(CompleteSectionSet)`
  only once all sections `0..=last_section_number` of the current version have
  arrived. Validates the long-form CRC before retaining bytes; restarts on a
  version bump.
- `CompleteSectionSet` — owns the original section bytes (so parsed views keep
  borrowing): generic `.table::<T>()` plus flattened logical helpers
  `.nit()` / `.bat()` / `.sdt()` / `.eit()`.
- `EitCollector` — EIT is complete only when every schedule `table_id` through
  `last_table_id` is done; sub-tables version independently, so per-`table_id`
  versions are exposed and EPG state can be pruned with `retain_logical` /
  `clear`.

`dvb-common`, `dvb-t2mi`, and `dvb-bbframe` are released in lockstep at 4.0.0
with no functional change (the t2mi crate-root diagram is updated to name
`AnyTableSection`).

# 6.3.0 — SCTE 35 crate, async streaming adapters, shared resync

A purely additive release. Two new crates and a shared TS-framing helper; no
breaking changes.

## `dvb-scte35` — ANSI/SCTE 35 2023r1 splice information (#58, lockstep, first release)

A spec-cited SCTE 35 **parser and symmetric serializer** (the differentiator vs
existing parse-only crates), with the workspace's byte-identical round-trip
discipline.

- `SpliceInfoSection` (0xFC): full header, encryption flags (encrypted body kept
  raw), 33-bit pts_adjustment, tier, descriptor loop, CRC_32.
- All six commands (splice_null/schedule/insert/time_signal/bandwidth_reservation/
  private_command) + splice_time/break_duration; `AnyCommand` dispatch + drift test.
- Splice descriptors (avail/DTMF/segmentation/time/audio) with typed
  `SegmentationTypeId`/`SegmentationUpidType`/`DeviceRestrictions`; raw fall-through;
  `AnySpliceDescriptor` dispatch + drift test.
- **Typed MPU/MID UPID sub-structures** (`Mpu`, `MidUpid` via `mpu()`/`mid()`).
- Decoded `Duration` accessors; 33-bit pts wrap handling.
- **Validated against all eight ANSI/SCTE 35 §14 worked sample messages**
  (parse → CRC → byte-identical round-trip → reparse). The spec transcription
  (`dvb-scte35/docs/scte_35.md`) was itself PDF-verified to 0 discrepancies.

## `dvb-stream` — async/tokio streaming adapters (#60, independent `0.1.0`)

`SectionStream` / `T2miEventStream` wrap `SiDemux` / `T2miPump` as
`futures_core::Stream`s over any `AsyncRead` (+ UDP-multicast constructor),
yielding owned events. No internal tasks; cancellation = drop. **Versioned and
released independently of the 6.x lockstep set** (tokio MSRV floats); not part of
the `v6.3.0` tag.

## `dvb-si` — shared TS resync helper (#61)

`dvb_si::ts::TsResync` (feature `ts`): stateful 188/204-byte byte-stream
resynchroniser — N-confirmation lock, Reed-Solomon 204-byte detection + parity
strip, resync/dropped-byte stats. `dvb-stream` and the `dvb-tools` CLIs now share
it instead of ad-hoc framing.

## Lockstep

The seven lockstep crates move to 6.3.0 (dvb-common, dvb-si, dvb-t2mi,
dvb-bbframe, dvb-conformance, dvb-tools, dvb-scte35). `dvb-stream` stays at
`0.1.0`. release.yml publishes in dependency order
(common → scte35 → bbframe → si → conformance → t2mi → tools); dvb-stream is
excluded from the lockstep tag.

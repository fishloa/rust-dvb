# 6.2.0 — two new crates: dvb-tools + dvb-conformance

A purely **additive** workspace minor — no breaking changes. It ships two new
published crates (an analyzer CLI and a TR 101 290 conformance monitor) and a
small `dvb-t2mi` convenience, all built on the existing library surface.

## `dvb-tools` — published analyzer binary (#59)

The former `si_dump` / `t2mi_dump` examples are promoted into one installable
binary crate (`cargo install dvb-tools`). Zero new external dependencies —
argument parsing stays on `std::env::args`. Five subcommands over aligned
188-byte `.ts` captures:

- **`dump <file.ts> [--json]`** — SI section dump (`SiDemux`), text or pretty
  JSON. (Absorbs `si_dump`.)
- **`services <file.ts>`** — SDT service tree with NIT logical-channel-number
  overlay (`SectionSetCollector` + `CompleteSdt`/`CompleteNit`).
- **`epg <file.ts> [--json]`** — `EpgStore` schedule; EIT events with service
  names attached from the SDT.
- **`pids <file.ts>`** — per-PID packet table + PCR-derived bitrate estimate.
- **`t2mi <file> [--pid 0xNNN|raw] [--inner] [--plp N]`** — T2-MI packet dump
  (absorbs `t2mi_dump`); `--inner` chain-unwraps to the inner MPEG-TS via
  `InnerTsRecovery` and writes the recovered packets to stdout, `--plp` filters
  one PLP. `dvb-tools t2mi … --inner | dvb-tools dump -` round-trips a T2-MI
  capture back to decoded SI.

The two absorbed examples (and their `[[example]]` manifest entries) are
removed; no duplicated logic.

## `dvb-conformance` — ETSI TR 101 290 stream monitor (#57)

A new crate: a feed-and-drain transport-stream conformance monitor
(`ConformanceMonitor::feed(packet, t) -> &[ConformanceEvent]`) on a
**caller-supplied-time model** — the crate never reads a clock; the caller
passes a monotonic `Duration` per packet. Every check cites its TR 101 290
v1.4.1 indicator; no magic numbers (PIDs/table_ids/timings are all named
constants).

- **Priority 1 (Table 5.0a):** `TS_sync_loss` (5-acquire/2-loss hysteresis),
  `Sync_byte_error`, `PAT_error_2`, `Continuity_count_error`, `PMT_error_2`,
  `PID_error`.
- **Priority 2 (Table 5.0b):** `Transport_error`, `CRC_error` (across
  PAT/CAT/NIT/SDT/BAT/EIT/TDT/TOT + PMT PIDs), `PCR_repetition_error` (>100 ms),
  `PCR_discontinuity_indicator_error`, `PTS_error` (>700 ms), `CAT_error`.
- **SI-repetition (Table 5.0c indicator 3.2):** maximum-interval checks for
  NIT_actual (10 s), SDT_actual (2 s), EIT-P/F actual (2 s), TDT (30 s).

**Honest about the model's limits** (documented in the crate README):
`PCR_accuracy_error` (2.4) is *not* implemented — its ±500 ns tolerance is not
computable from coarse caller timestamps (it needs hardware arrival times). The
25 ms minimum-gap dimension of 3.2 is deferred — it needs per-section tracking
to avoid false positives on dense multi-section tables. Entirely-absent tables
are the domain of the presence indicators 3.1/3.5/3.6/3.8, not 3.2.

48 unit tests (trip + compliant per indicator) plus fixture smoke tests.

## `dvb-t2mi` — per-PLP inner-TS filter (#146)

`InnerTsRecovery::new_for_plp(t2mi_pid, plp_id)` recovers the inner TS for a
single baseband-frame PLP only; `filtered_bbframes()` reports how many BBFrames
were skipped. `new()` is unchanged.

## Lockstep

All six crates move to 6.2.0 (`dvb-common` and `dvb-bbframe` are version-only).
`release.yml` publishes in topological order: common → bbframe → si →
conformance → t2mi → tools.

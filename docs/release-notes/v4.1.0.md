# 4.1.0 — decoded accessors for BCD/MJD wire fields

A purely additive release: typed, decoded getters and symmetric `set_*`
encoders so consumers stop hand-decoding the BCD/MJD numbers DVB packs into the
wire. No wire-format or existing-API change. Closes #37 and #38.

## Shared codec in `dvb-common`

The BCD/MJD logic that was copy-pasted inside `dvb-si` now lives once in
`dvb-common`:

- `dvb_common::bcd` — binary-coded-decimal encode/decode, **dependency-free**.
- `dvb_common::time` — a BCD `HHMMSS` duration codec (dependency-free) plus the
  MJD↔calendar UTC codec (EN 300 468 Annex C) behind a new optional **`chrono`**
  feature. `dvb-common`'s default build stays zero-dependency.

## Decoded accessors in `dvb-si`

Every site that exposed a raw `*_bcd` / `*_raw` field now also offers a decoded
getter and a `set_*` encoder (setters return `Result`; the new
`Error::ValueOutOfRange` covers values that don't fit the field):

- **Time:** `EitEvent::duration()` (the #37 ask) and `start_time()`;
  `TotSection::utc_time()` (it had the raw bytes but no decoder, unlike TDT);
  `TdtSection::set_utc_time()`; `CompleteEitEvent` getters.
- **Delivery descriptors (the #38 ask, plus cable/terrestrial):** satellite and
  cable `frequency_hz()` / `symbol_rate_sps()`, satellite
  `orbital_position_deg()`, terrestrial `centre_frequency_hz()`.
- **Other BCD/packed fields:** `FrequencyListDescriptor::centre_frequencies_hz()`,
  `LocalTimeOffsetEntry` offsets + `time_of_change()`, and the `PdcDescriptor`
  PIL components.

Frequencies are exposed in **Hz** (`u64`) rather than kHz: cable (100 Hz) and
terrestrial (10 Hz) are finer than 1 kHz, so an integer kHz would silently
round.

## `dvb-t2mi`

The crate-root quickstart doctest is now gated on the `ts` feature, so
`cargo test --no-default-features --doc` compiles cleanly.

All four crates are released in lockstep at 4.1.0.

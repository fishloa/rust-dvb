# 6.5.0 — SI typed-decode completeness

A purely **additive** release — typed decoders for the last few raw tails plus
decoded-value accessors. No breaking changes.

## `dvb-t2mi` — decoded emission-time accessors (#47)

`T2TimestampPayload` gains computed accessors over its raw fields (TS 102 773
§5.2.7 / Table 4):

- `Bandwidth::t_sub()` — the subsecond unit `T_sub` as an exact rational
  `(1000, D)` ns (= 1/D µs; D ∈ {131,40,48,56,64,80}), no floating point.
- `emission_offset()` — seconds + subsecond nanoseconds since the 2000 epoch
  (pre-UTCO).
- `emission_time_utc()` / `set_emission_time_utc()` — `chrono`-gated
  `DateTime<Utc>` with UTCO leap-second correction. A new **default-OFF
  `chrono` feature** is added to `dvb-t2mi`, mirroring `dvb-si`.
- `is_null()` / `is_relative()` and symmetric `set_*` encoders.

`dvb-common::time` gains `SECS_2000_EPOCH` + 2000-epoch encode/decode helpers
(next to the existing MJD/BCD codecs).

## `dvb-si` — SSU chain completion (#50)

The DVB-SSU receiver chain is now end-to-end typed (additive):

- `carousel::GroupInfoIndication` + `GroupInfo` — typed parse/serialize of the
  SSU DSI `privateData` field (TS 102 006 §8.1.1 Table 6).
- `DataBroadcastIdDescriptor::id_selector_decoded()` — decode-on-demand accessor
  returning a typed `IdSelector` (`Ssu` for `data_broadcast_id = 0x000A`, TS
  102 006 §7.1 Table 4; `Raw` otherwise). The `id_selector` field stays
  `&[u8]` — byte-identical round-trip, **zero regression for non-SSU callers**.
- `SsuIdSelector` / `SsuOuiEntry` / `IdSelector` types + `DATA_BROADCAST_ID_SSU`.

## `dvb-si` — S2Xv2 satellite delivery (#53)

- `S2Xv2SatelliteDeliverySystem` — typed body for extension descriptor `0x24`
  (EN 300 468 Tables 144a–144c, §6.4.6.5.3): all conditional blocks typed
  (scrambling index for modes 1/2, timeslice for modes 2/5, channel-bond loop,
  the superframe block for modes 4/5), with the trailing reserved bytes
  preserved verbatim. Closes the extension-descriptor (0x7F) typing sweep; the
  remaining `Raw` bodies are those whose specs are not vendored (CPCM, DTS
  family, CI-ancillary, …).

## Notes

- `SAT (0x4D)` variant bodies (roadmap #52) were already fully typed since 6.0.x;
  that issue was closed as already-complete.
- Lockstep across the seven core crates. `dvb-stream` stays `0.1.0` (independent
  cadence) and is not part of this set.

# 4.3.0 — decoded accessors & analysis building blocks

An additive release across the workspace: new convenience/analysis layers and
decoded accessors, one performance change, one internal cleanup. No breaking
changes. Lockstep version bump 4.2.0 → 4.3.0 (dvb-common has no code change).

## dvb-si

- **`epg` module (feature `chrono`)** — `EpgStore`, an electronic-programme-guide
  layer over `EitCollector`. Keyed by `(original_network_id,
  transport_stream_id, service_id)`, it keeps a deduplicated, time-ordered event
  list and decodes the fields callers actually want: short-event name/text,
  **extended-event text concatenated across fragments per EN 300 468 §6.2.15**,
  content genre, parental ratings, and CRIDs. `now_and_next(key, at)` returns the
  on-air and next events; `feed_sdt` joins service names from the SDT;
  `services()` enumerates cached services; `retain_services`/`clear` bound
  long-running memory. Serialize-only serde export. (#51)
- **`resync` module** — `TsResync` recovers 188-byte TS packet alignment from an
  arbitrary byte stream (junk prefixes, mid-stream loss) and detects/strips
  204-byte Reed-Solomon packets, with resync/dropped-byte stats. (#61)
- **Typed adaptation field + PCR** — `TsPacket::adaptation_field()` decodes the
  flags, PCR/OPCR (`Pcr` with `as_27mhz()`), and splice countdown per
  ISO/IEC 13818-1:2007 §2.4.3.4; previously the field was skipped. (#48)
- **Perf:** `SiDemux::feed` now does a single PID-map lookup per packet. (#44)
- **Cleanup:** removed the dead write-only `expected` field in
  `SectionReassembler`. (#46)

## dvb-t2mi

- **Decoded T2-MI timestamp accessors** — `T2TimestampPayload::emission_offset`
  (time since the 2000 epoch as `core::time::Duration`), the per-bandwidth
  `subseconds_per_second` units (TS 102 773 §5.2.7 Table 4), and
  `is_null`/`is_relative`. Civil-UTC conversion (the `utco` leap-second offset)
  is intentionally deferred. (#47)

## dvb-bbframe

- **Buffer-reusing extraction** — `CarryOverExtractor::feed_hem_into` /
  `feed_nm_into` append completed TS packets into a caller-provided `Vec`
  (reusable across frames), avoiding the per-frame allocation of
  `feed_hem`/`feed_nm`. (#45)

## dvb-common

Version-lockstep only; no code change.

## Validation

Two real broadcast captures were added to the test suite (a stripped Capital TV
Colombia T2-MI stream and the existing satellite SI capture) to validate the
T2-MI timestamp and EPG paths on real data rather than synthetic vectors.
